# Itinerary Mate Product Brief

## Register

Product UI. The design serves planning, editing, and travel-day use.

## Product Purpose

Itinerary Mate is a local-first trip workspace for planning, carrying, and reviewing personal travel. It should help the traveler answer four questions quickly:

- What is happening next?
- Where is it?
- What does it cost?
- What still needs attention?

## Primary Users

- A detail-oriented traveler planning multi-city trips.
- A traveler on the road using a phone to check the next stop, map link, booking, or expense.
- A future self reviewing where the trip went and what it cost.

## Product Principles

- Action before archive: show the next useful action before source history or imported metadata.
- Edit in place: adding places, costs, notes, bookings, and status should feel close to the object being edited.
- Fewer words, clearer controls: labels should be short and concrete.
- Real images only: when a card needs photography, use a saved real image. If none exists, show a compact missing-image state instead of fake scenic art.
- Map and list stay connected: day plans, routes, and map pins should reinforce each other.
- Local-first honesty: saved locally, offline basics, and non-live data should be clear without dominating the interface.

## Key Workflows

- Pick a trip and see the active plan.
- Add a place, note, meal, transfer, or rest block.
- Track an expense or update a planned cost.
- Review a day by timeline, map, and pacing.
- Open the next map link or booking detail quickly.
- Confirm or clean up imported/actual trip records.

## Copy Tone

Plain, travel-useful, and concise. Use words like `Plan`, `Map`, `Budget`, `Bookings`, `Add place`, `Add expense`, `Needs attention`, `Saved locally`.

Avoid command-center language, source-system jargon, repeated imported counts, and labels that explain implementation details.
